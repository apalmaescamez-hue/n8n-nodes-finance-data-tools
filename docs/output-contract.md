# Output contract

All nodes return one item whose `json` is a standard envelope. This makes workflow routing, AI Agent summarization, and tests consistent across the package.

## Standard envelope

```json
{
  "success": true,
  "operation": "operation_name",
  "data": {},
  "metadata": {
    "generatedAt": "2026-05-09T00:00:00.000Z",
    "durationMs": 12,
    "rowCount": 10,
    "columnCount": 4
  },
  "warnings": [
    {
      "code": "WARNING_CODE",
      "severity": "warning",
      "message": "Human-readable warning.",
      "field": "optionalField",
      "details": {}
    }
  ],
  "errors": [],
  "auditTrail": [
    {
      "timestamp": "2026-05-09T00:00:00.000Z",
      "step": "operation_completed",
      "message": "Operation completed.",
      "details": {}
    }
  ]
}
```

## Field semantics

| Field | Meaning |
| --- | --- |
| `success` | `true` when no blocking errors were produced. `false` when blocking validation or execution errors exist. |
| `operation` | Stable operation identifier executed by the node or facade. |
| `data` | Node-specific result. Can be `null` on failure. |
| `metadata.generatedAt` | ISO timestamp generated by the domain service. |
| `metadata.durationMs` | Approximate duration in milliseconds. |
| `metadata.rowCount` | Number of input rows/items considered by the operation. |
| `metadata.columnCount` | Estimated number of input columns/fields. |
| `warnings` | Non-blocking issues that users and AI Agents must mention when relevant. |
| `errors` | Blocking issues or child-operation errors. |
| `auditTrail` | Deterministic trace of major processing steps. |

## Warning and error shape

```json
{
  "code": "MISSING_FIELD",
  "severity": "warning",
  "message": "A configured field was missing.",
  "field": "revenue",
  "details": { "rowIndex": 2 }
}
```

Errors use the same shape with `severity: "error"`.

## Predictive Analytics contract

Predictive operations return `data` shaped like this:

```json
{
  "operation": "moving_average_forecast",
  "valueColumn": "revenue",
  "xColumn": null,
  "currency": "EUR",
  "model": {
    "slope": "100",
    "intercept": "900",
    "rSquared": "0.95",
    "observationCount": 6,
    "xColumn": "periodIndex",
    "yColumn": "revenue"
  },
  "forecast": [
    {
      "step": 1,
      "forecast": "1300",
      "lowerBound": "1200",
      "upperBound": "1400",
      "method": "moving_average_forecast"
    }
  ],
  "metrics": {
    "mae": "10",
    "mse": "100",
    "rmse": "10",
    "mape": "2.5",
    "zeroActualCount": 0
  },
  "summary": "Directional forecast summary.",
  "limitations": [
    "Forecasts are directional and should not be interpreted as certainty."
  ],
  "inputSummary": {
    "rowCount": 6,
    "validObservationCount": 6,
    "ignoredObservationCount": 0,
    "horizon": 3
  }
}
```

`model` is present only when the selected predictive method fits an explicit model. Metrics can be `null` where fitted values are not available or where division by zero would make the metric misleading.

## AI Finance Tool contract

The AI facade wraps the selected child operation:

```json
{
  "requestedOperation": "forecast_financial_metric",
  "executedOperation": "moving_average_forecast",
  "result": {},
  "childSuccess": true,
  "childMetadata": {},
  "controls": {
    "allowPredictiveOperations": true,
    "maxForecastHorizon": 12,
    "maxRows": 1000,
    "rowCount": 6
  },
  "summary": "AI Finance Tool executed the operation successfully.",
  "limitations": [],
  "agentInstructions": {
    "canSummarize": true,
    "mustMentionWarnings": true,
    "mustNotOverstatePredictions": true,
    "mustNotAssumeCausality": true,
    "recommendedPrompt": "Summarize the forecast..."
  }
}
```

Agents should summarize `data.result`, not invent calculations outside the envelope.
